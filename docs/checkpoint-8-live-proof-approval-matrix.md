# Checkpoint 8 Live Proof Approval Matrix

## Scope

This is the authoritative Checkpoint 8 read-only discovery and approval matrix
for live UiPath execution in `TreatmentAccessHackathon`.

Discovery order follows the project rule:

1. Login, organization, tenant, and folder.
2. Product/tool registration and command discovery.
3. Activation and permissions through read-only lists.
4. Runtime readiness only after the feature surface is present.

No live UiPath resource was created, updated, deleted, deployed, activated,
started, assigned, completed, or otherwise mutated during this lane.

## Stop Conditions

Stop and return to the orchestrator before execution if any of these are true:

- Active org/tenant/folder is not `galacticus` / `DefaultTenant` /
  `TreatmentAccessHackathon`.
- The next step would create, update, delete, deploy, activate, start, assign,
  complete, or submit anything live without explicit approval.
- A required UiPath surface is absent after registration/discovery and official
  activation checks.
- H1 cannot be achieved because Data Fabric/API Workflow/agent permissions are
  unavailable.
- H2 cannot be achieved because Action Center task creation or completion is
  unavailable.
- H3 cannot be achieved because no runtime/process/package/job path is
  available after publish/deploy approval.
- Any step needs real PHI, real payer/provider credentials, or autonomous
  medical/legal advice.
